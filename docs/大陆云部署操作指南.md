# 大陆云部署操作指南

本文档用于把现场检查调度系统部署到阿里云、腾讯云或华为云的中国大陆节点，目标是支持中国内地用户稳定访问。

## 一、推荐结论

当前产品是一个 Next.js 全栈应用，API 已内置在 Web 应用中。第一阶段建议采用：

```text
大陆云 ECS/CVM/HECS 单机 + Docker + Nginx + HTTPS + 已备案域名
```

原因：

- 三家云都支持，迁移成本低。
- 最适合当前 demo/试运行阶段。
- 可以先单机运行，后续再平滑迁移到容器服务或 Kubernetes。
- 不依赖 Vercel、Cloudflare Workers、境外网络。

推荐优先级：

1. 如果已有阿里云账号或域名：优先阿里云 ECS。
2. 如果已有腾讯云备案资源：优先腾讯云 CVM / 轻量应用服务器。
3. 如果公司云资源在华为云：使用华为云 HECS / ECS。

三家云的技术部署方式一致：拉取 GitHub 仓库，构建 Docker 镜像，启动容器，再用 Nginx 绑定域名和 HTTPS。

## 二、备案前提

只要域名解析到中国大陆服务器，就必须完成 ICP 备案。

官方口径：

- 阿里云：域名解析到阿里云中国内地节点服务器，必须完成 ICP 备案。
- 腾讯云：未备案域名直接解析到腾讯云中国境内云服务器，会被识别并阻断访问。
- 华为云：使用中国大陆节点服务器提供互联网信息服务，需要提交备案申请。

因此正式上线前需要：

1. 自有域名，例如：

```text
inspection.example.com
```

2. 大陆云服务器。
3. ICP 备案号。
4. 页面底部展示备案号。
5. 如正式对公众开放，按要求完成公安联网备案。

## 三、服务器建议

试运行配置：

```text
2 vCPU / 4 GB RAM / 40 GB 云盘
操作系统：Ubuntu 22.04 LTS
带宽：3 Mbps 起
端口：80、443、22
```

生产试点配置：

```text
2-4 vCPU / 8 GB RAM / 80 GB 云盘
带宽：5-10 Mbps
建议开启云监控、快照和安全组白名单
```

当前应用使用内存态工作区，服务器重启后会回到内置测试数据。正式生产前应接入数据库。

## 五、服务器部署步骤

以下步骤适用于阿里云 ECS、腾讯云 CVM、华为云 ECS/HECS。

### 1. 安装基础环境

```bash
sudo apt update
sudo apt install -y ca-certificates curl git nginx
```

安装 Docker：

```bash
curl -fsSL https://get.docker.com | sudo sh
sudo usermod -aG docker $USER
```

重新登录服务器后检查：

```bash
docker version
docker compose version
```

### 2. 拉取代码

```bash
git clone https://github.com/Vixeruntz/onsite-inspection-scheduler.git
cd onsite-inspection-scheduler
```

如果仓库保持 private，需要先配置 GitHub Deploy Key 或 Personal Access Token。

### 3. 启动服务

```bash
docker compose up -d --build
```

查看状态：

```bash
docker ps
docker logs -f onsite-inspection-scheduler
```

健康检查：

```bash
curl http://127.0.0.1:3000/api/health
```

### 4. 配置 Nginx

将 `inspection.example.com` 替换为正式域名。

```nginx
server {
    listen 80;
    server_name inspection.example.com;

    client_max_body_size 20m;

    location / {
        proxy_pass http://127.0.0.1:3000;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
    }
}
```

保存为：

```text
/etc/nginx/sites-available/onsite-inspection-scheduler
```

启用配置：

```bash
sudo ln -s /etc/nginx/sites-available/onsite-inspection-scheduler /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl reload nginx
```

### 5. 配置 HTTPS

如果使用云厂商证书服务：

- 在云厂商申请免费或企业 SSL 证书。
- 将证书绑定到负载均衡或 Nginx。

如果使用 Certbot：

```bash
sudo apt install -y certbot python3-certbot-nginx
sudo certbot --nginx -d inspection.example.com
```

## 六、云厂商差异

### 阿里云

推荐组合：

```text
ECS + 云解析 DNS + SSL 证书服务 + ICP 备案
```

可升级：

```text
ACR 镜像仓库 + SAE / ACK
```

适合已有阿里云备案或企业资源的情况。

### 腾讯云

推荐组合：

```text
CVM / 轻量应用服务器 + DNSPod + SSL 证书 + ICP 备案
```

可升级：

```text
TCR 镜像仓库 + TKE / 云托管
```

适合已有腾讯云备案、企业微信生态或微信登录需求的情况。

### 华为云

推荐组合：

```text
HECS / ECS + 云解析 DNS + SSL 证书 + ICP 备案
```

可升级：

```text
SWR 镜像仓库 + CCE
```

适合公司已有华为云资源或政企客户场景。

## 七、上线验收

上线前检查：

```bash
curl https://inspection.example.com/api/health
curl https://inspection.example.com/api/backend/workspace
```

浏览器检查：

- 首页能打开。
- 项目维护、人员维护、规则维护、排期方案、导出中心都能进入。
- 导出 Excel 能下载。
- 排期项目详情能打开五步业务解释。
- 页面底部展示 `Created by Hank Zhao` 和 ICP 备案号。

网络检查：

- 中国移动网络访问。
- 中国联通网络访问。
- 中国电信网络访问。
- 公司内网访问。
- 手机热点访问。

## 八、后续生产化事项

当前部署适合 demo 和试运行。正式生产建议补齐：

1. PostgreSQL 或 MySQL 持久化。
2. 对象存储保存导出文件。
3. 企业登录：飞书、企业微信、钉钉或公司 SSO。
4. 操作审计日志。
5. 自动备份。
6. HTTPS 证书自动续期。
7. 云监控和异常告警。
8. 灰度发布和回滚机制。

## 九、仍需用户提供

继续部署到大陆云前，需要确认：

1. 选择哪一家云：阿里云、腾讯云、华为云。
2. 是否已有备案域名。
3. 服务器所在地域。
4. 是否能提供云服务器 SSH。
5. GitHub private 仓库是否允许服务器拉取。
6. 是否需要先改为 public 或配置 Deploy Key。
