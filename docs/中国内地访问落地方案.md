# 中国内地访问落地方案

## 当前状态

产品已部署到 Cloudflare Workers：

```text
https://onsite-inspection-scheduler.zlonicera.workers.dev
```

该地址属于 Cloudflare 全球网络入口，可以用于海外访问、临时演示和技术验证，但不能保证中国内地稳定访问。

当前 Cloudflare 账户状态：

- 已有 Worker：`onsite-inspection-scheduler`
- 已启用 `workers.dev` 入口
- 账号级 workers.dev 子域名：`zlonicera`
- Cloudflare 账户中尚未接入自有域名

## 结论

如果目标是中国内地正式访问，应采用以下两条路径之一：

1. 推荐正式路径：自有域名 + ICP 备案/许可证 + Cloudflare China Network。
2. 快速大陆路径：部署到阿里云、腾讯云、华为云等中国大陆云服务，并完成 ICP 备案。

普通 `workers.dev` 地址、普通 Cloudflare 全球网络、自有域名但未备案，都不能作为中国内地正式稳定入口。

## 路径一：Cloudflare China Network

适合场景：

- 希望继续使用 Cloudflare 的全球网络、安全能力和 Workers/Assets。
- 用户既有中国内地用户，也有境外用户。
- 公司可以完成 ICP 备案，并能开通 Cloudflare China Network。

前置条件：

1. 拥有自有域名，例如：

```text
inspection.example.com
```

2. 域名完成 ICP 备案或 ICP 许可证。
3. Cloudflare 账户开通 China Network。
4. 域名接入 Cloudflare，并按 China Network 要求配置中国网络。
5. 首页底部展示 ICP 备案号。

执行步骤：

1. 确认正式域名。
2. 在中国大陆云服务商处完成 ICP 备案。
3. 在 Cloudflare 添加该域名为 Zone。
4. 将当前 Worker 绑定到正式域名路由。
5. 通过 Cloudflare China Network 开通中国内地网络能力。
6. 在产品底部加入备案号，例如：

```text
京ICP备XXXXXXXX号
```

7. 从中国移动、中国联通、中国电信网络分别验证访问。

风险与限制：

- Cloudflare China Network 不是普通免费 Workers 部署能力，需要单独开通。
- 未备案域名无法作为中国内地正式网站入口。
- 如果后续接入数据库或外部 API，动态请求仍需评估跨境延迟和合规。

## 路径二：大陆云服务部署

适合场景：

- 主要用户都在中国内地。
- 希望最快获得稳定访问。
- 可以接受把生产环境部署到中国大陆云服务。

推荐部署形态：

```text
用户浏览器
  -> 已备案域名
  -> 大陆 CDN / 负载均衡
  -> Node.js / 容器服务
  -> 数据库 / 对象存储
```

可选平台：

- 阿里云：ECS / SAE / 函数计算 / CDN
- 腾讯云：CVM / CloudBase / EdgeOne / CDN
- 华为云：ECS / FunctionGraph / CDN

执行步骤：

1. 购买或确认自有域名。
2. 选择备案主体：公司或个人。
3. 完成 ICP 备案。
4. 部署 Node.js 服务。
5. 配置 HTTPS 证书。
6. 将域名解析到大陆云服务。
7. 页面底部展示备案号。
8. 做三网访问测试和导出 Excel 测试。

优势：

- 中国内地访问稳定性更可控。
- 备案和云服务接入流程更标准。
- 后续接入企业登录、数据库、文件存储更直接。

代价：

- 需要独立运维大陆云资源。
- 与当前 Cloudflare/Vercel 的一键部署模式不同。

## 推荐决策

如果只是给少量内部用户演示：

```text
继续使用 Cloudflare Workers 或 Vercel，配合公司网络/VPN。
```

如果要给中国内地客户正式试用：

```text
优先走大陆云服务部署，并完成 ICP 备案。
```

如果产品未来要同时服务国内外用户：

```text
使用自有域名，国内走 Cloudflare China Network 或大陆云，海外保留 Cloudflare/Vercel。
```

## 需要用户提供的信息

要继续执行中国内地正式访问落地，需要以下信息：

1. 正式访问域名。
2. 域名是否已有 ICP 备案。
3. 备案主体：公司或个人。
4. 是否已有大陆云账号。
5. 是否希望继续使用 Cloudflare China Network。
6. 是否需要接入企业登录，例如飞书、企业微信、钉钉或公司 SSO。

## 当前可以立即做的技术动作

在拿到域名和备案信息之前，当前可继续完成：

- 保持 Cloudflare Worker 全球访问版本。
- 将 GitHub 仓库作为 Lovable/Cloudflare/Vercel 的统一源码。
- 为未来备案号预留页脚展示位置。
- 准备大陆云部署配置。
- 准备企业登录方案。

## 当前不能直接完成的事项

以下事项无法在没有域名和备案信息的情况下自动完成：

- 开通中国内地正式访问入口。
- 绑定已备案域名。
- 进入 Cloudflare China Network。
- 展示真实 ICP 备案号。
- 保证大陆三网稳定访问。
